# Dog Donations & Redesign — Design Spec

## Overview

Полный редизайн фронтенда "I Love Dogs" с фокусом на адресные пожертвования конкретным собакам и бронирование прогулок. Сайт должен выглядеть живым и человечным, а не сгенерированным ИИ.

**Тон:** тёплый и эмоциональный — истории собак от волонтёров, фото до/после, личные тексты. Как будто сайт ведёт реальный человек.

**Визуальный стиль:** чистый современный — белый фон, чёрный текст, много воздуха, sans-serif (Inter). Как Notion или Medium. Контент на первом месте.

**Бэкенд:** NestJS API на `http://localhost:3000/api/v1`, интеграция реальная (не мок).

## Что убираем

- Glassmorphism-эффекты (backdrop-filter, blur)
- Анимированные блобы на главной
- Gradient-текст
- Тёмную тему (откладываем)
- Все стоковые фразы и корпоративный тон

## Визуальная система

### Палитра

| Token | Значение | Использование |
|-------|----------|---------------|
| `--text-primary` | `#1a1a1a` | Основной текст |
| `--text-secondary` | `#666` | Вторичный текст |
| `--text-muted` | `#999` | Мета-информация, подписи |
| `--bg-primary` | `#fff` | Фон страницы |
| `--bg-secondary` | `#f7f7f7` | Фон карточек/секций |
| `--border` | `#eee` | Границы, разделители |
| `--accent` | `#1a1a1a` | CTA-кнопки (чёрные с белым текстом) |
| `--accent-secondary` | `transparent` + `border: #ddd` | Вторичные кнопки |

### Типографика

- Шрифт: Inter (уже подключен, с Cyrillic)
- Заголовки: font-weight 700, без uppercase
- Текст: font-weight 400, line-height 1.7
- Мета-инфо: font-size 11-12px, uppercase, letter-spacing 2-3px, color `--text-muted`

### Компоненты

- Кнопки: скруглённые (border-radius 8px), без теней. Primary — чёрный фон/белый текст. Secondary — прозрачный с border.
- Карточки: белый фон, тонкая граница `#eee`, без теней. Hover — лёгкая тень или border чуть темнее.
- Инпуты: простые, border `#ddd`, без иконок внутри.
- Бейджи: маленькие, скруглённые, нейтральные цвета.
- Прогресс-бар: тонкий, чёрная полоса на сером фоне.

## Страницы

### 1. Главная `/`

**Убираем:** hero с блобами, gradient-текст, анимации.

**Вместо этого:**
- Короткий заголовок + подзаголовок живым, тёплым языком (не стоковые фразы)
- Сетка из 3-4 карточек активных собак — данные из `GET /dogs?limit=4`
- Секция "Как это работает" — 3 шага текстом, без иконок

### 2. Каталог собак `/dogs`

**Фильтры:** строка сверху — город, пол, порода. Селекты/инпуты в ряд. Используют query-параметры API (`city`, `gender`, `breed`).

**Карточки собак:**
- Большое фото сверху (`cover_photo_url`)
- Имя, порода, возраст, город — коротко, фактами
- Если есть активный сбор — маленький прогресс-бар
- По клику — переход на `/dogs/:id`
- Пагинация: `limit`/`offset` из API

**Тон:** никаких "Подарите шанс!" — просто "Рекс, метис, 2 года, Москва". Фото делает эмоциональную работу.

**API:** `GET /dogs?city=...&gender=...&breed=...&limit=20&offset=0`

### 3. Страница собаки `/dogs/:id`

Единая длинная страница, скролл сверху вниз. Пять секций:

#### 3.1 Герой
- Большое фото на всю ширину — `cover_photo_url`
- Имя собаки крупным шрифтом
- Мета-инфо: порода, возраст, пол, город
- Имя куратора (без ссылки — страница куратора не в скоупе)

**API:** `GET /dogs/:id`

#### 3.2 История
- `description` из модели Dog — текст от куратора
- Без заголовка "История" — просто текст, как пост в блоге
- Тон задаётся контентом, не дизайном

#### 3.3 Чем помочь (Goals)
- Заголовок: "Чем помочь" или "Сейчас нужно"
- Карточки целей: категория (бейдж), название, описание, прогресс-бар (`amount_collected`/`amount_target`), дедлайн
- Кнопка "Помочь" на каждой карточке — открывает модалку пожертвования

**API:** `GET /goals/dog/:dogId`

#### 3.4 Отчёты
- Лента обновлений от куратора
- Каждый отчёт: дата, тип (бейдж), текст, фото
- Хронологический порядок, новые сверху
- Создаёт прозрачность — видно куда пошли деньги

**API:** `GET /reports/dog/:dogId`

#### 3.5 Прогулки
- Заголовок: "Погулять с [имя]"
- Форма бронирования: дата, длительность (30/60/90 мин), заметки
- Требует авторизации. Если не авторизован — "Войдите чтобы записаться"
- Список ближайших прогулок — видно что собаку выгуливают

**API:**
- `POST /walks` (бронирование)
- `GET /walks/dog/:dogId` (список прогулок)

### 4. Модалка пожертвования

Открывается по кнопке "Помочь" на карточке цели.

**Авторизованный пользователь:**
- Показываем название цели ("Лечение лапы Барона")
- Поле суммы (свободная, мин. 10 руб.)
- Кнопка "Пожертвовать"
- **API:** `POST /payments/donate` с `{ amount, goal_id }`

**Неавторизованный пользователь:**
- То же + поля email и display_name (опционально)
- **API:** `POST /payments/donate-anon` с `{ amount, goal_id, email, display_name }`

**Подтверждение:** простое сообщение "Готово. 500 руб. на лечение Барона." Без попапов "Спасибо за щедрость!"

### 5. Авторизация `/auth/login`, `/auth/register`

- Существующая логика остаётся (JWT, AuthContext, localStorage)
- Обновляем визуал под новый стиль: белый фон, чёрные кнопки, убираем glassmorphism
- После логина — редирект обратно на страницу, с которой пришёл (а не на dashboard)

### 6. Хедер

- Логотип (текст) слева
- Навигация: Собаки, О нас
- Войти/Профиль справа
- Простой, без теней и blur

### 7. Футер

- Минимальный: копирайт, ссылки на соцсети, контакты

## Навигация

- Главная → каталог → страница собаки — линейный путь
- На карточке в каталоге — клик ведёт на `/dogs/:id`
- Хлебные крошки на странице собаки: Собаки > Барон

## Что НЕ делаем в первой итерации

- Личный кабинет / профиль пользователя
- Страницу куратора (`/curators/:id`)
- Опекунство (patronage)
- Страницы "О нас", "Контакты" (только ссылки в навигации)
- Тёмную тему
- Настройку платёжного провайдера (фронтенд вызывает API, бэкенд сам работает с YooKassa)

## Технические решения

- **Фреймворк:** Next.js 16 (App Router), React 19
- **Стили:** CSS Modules (сохраняем подход), обновляем globals.css с новыми CSS-переменными
- **HTTP:** Axios (уже настроен с интерцепторами)
- **Состояние:** AuthContext (уже есть), серверные компоненты где возможно
- **Роутинг:** App Router — `/dogs/[id]/page.tsx` для динамической страницы
